# GREP — Réponses Round 1

## Niveau 1 — Basique

```bash
# 1. Lignes contenant "erreur" (insensible à la casse)
grep -i "erreur" grep_exo.txt

# 2. Lignes contenant [ERROR] ou [FATAL]
grep -E "\[ERROR\]|\[FATAL\]" grep_exo.txt
# ou en BRE :
grep "\[ERROR\]\|\[FATAL\]" grep_exo.txt

# 3. Nombre de lignes contenant INFO
grep -c "INFO" grep_exo.txt

# 4. Lignes ne contenant PAS de #
grep -v "#" grep_exo.txt

# 5. Numéro de ligne de chaque occurrence de alice
grep -n "alice" grep_exo.txt

# 6. Lignes commençant par un chemin /xxx
grep -E "^/[a-z]" grep_exo.txt
```

## Niveau 2 — Regex et options

```bash
# 7. Extraire uniquement les adresses IP
grep -Eo "[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}" grep_exo.txt

# 8. Lignes commençant par une date YYYY-MM-DD
grep -E "^[0-9]{4}-[0-9]{2}-[0-9]{2}" grep_exo.txt

# 9. Lignes se terminant par "bash"
grep "bash$" grep_exo.txt

# 10. Lignes contenant un mot de 4 lettres exactement
# ⚠️ [a-Z] est invalide, toujours utiliser [a-zA-Z] ou [[:alpha:]]
grep -Ew "[a-zA-Z]{4}" grep_exo.txt

# 11. URLs commençant par https
grep "https://" grep_exo.txt

# 12. Codes HTTP 4xx
# ⚠️ -w ne fonctionne pas ici, les codes ne sont pas des "mots" POSIX
grep -Eo " 4[0-9]{2} " grep_exo.txt
```

## Niveau 3 — Contexte et combinaisons

```bash
# 13. 2 lignes après chaque [ERROR]
grep -A 2 "\[ERROR\]" grep_exo.txt

# 14. 1 ligne avant ET après chaque [FATAL]
grep -B 1 -A 1 "\[FATAL\]" grep_exo.txt
# ou :
grep -C 1 "\[FATAL\]" grep_exo.txt

# 15. Chercher WARNING récursivement dans /var/log/
sudo grep -r "WARNING" /var/log/

# 16. Lignes contenant "serveur" mais pas "redémarr"
grep "serveur" grep_exo.txt | grep -v "redémarr"

# 17. Extraire les adresses email
grep -Eo "[a-zA-Z0-9._%+-]+@[a-zA-Z0-9.-]+\.[a-z]{2,}" grep_exo.txt

# 18. Lignes contenant exactement le mot "erreur"
grep -w "erreur" grep_exo.txt
```

## Niveau 4 — Fichiers système

```bash
# 19. Utilisateurs avec /bin/bash
grep -F "/bin/bash" /etc/passwd

# 20. Utilisateurs avec UID > 1000
grep -E "^[^:]+:[^:]+:[0-9]{4,}" /etc/passwd

# 21. Comptes sans répertoire home dans /home
grep -v "/home" /etc/passwd

# 22. Afficher uniquement les noms d'utilisateurs (premier champ)
grep -oE "^[a-z_-]+" /etc/passwd

# 23. Tentatives SSH échouées
sudo grep -E "Failed|Invalid" /var/log/secure
# ou sur Debian/Ubuntu :
sudo grep -E "Failed|Invalid" /var/log/auth.log

# 24. Fichiers dans /etc/ contenant 192.168
sudo grep -r "192.168" /etc/

# 25. Lignes non vides et non commentées dans /etc/fstab
grep -Ev "^$|^#" /etc/fstab
```

## Niveau 5 — Pièges et cas tordus

```bash
# 26. Chercher "3.14159" littéralement (. n'est pas un joker)
grep -F "3.14159" grep_exo.txt

# 27. Chercher "(test)" littéralement
# ⚠️ avec -F tout est littéral, pas besoin d'échapper
grep -F "(test)" grep_exo.txt

# 28. Chercher ERROR et WARNING avec -e
grep -e "ERROR" -e "WARNING" grep_exo.txt

# 29. Lignes contenant le mot "le" au moins 2 fois
grep -P "(\ble\b.*){2}" grep_exo.txt

# 30. Fichier + numéro de ligne contenant "root" sans le contenu
grep -n "root" /etc/passwd | cut -d: -f1,2
```
